1. **Create migration `supabase/migrations/20260316000000_add_deletion_columns.sql`:**
   ```sql
   ALTER TABLE public.profiles ADD COLUMN IF NOT EXISTS deletion_scheduled_at timestamptz DEFAULT NULL;

   -- Cleanup function: delete auth users whose grace period has expired
   CREATE OR REPLACE FUNCTION public.cleanup_expired_deletions()
   RETURNS void AS $$
   DECLARE expired_user RECORD;
   BEGIN
     FOR expired_user IN
       SELECT id FROM public.profiles
       WHERE deletion_scheduled_at IS NOT NULL AND deletion_scheduled_at <= now()
     LOOP
       DELETE FROM auth.users WHERE id = expired_user.id;
     END LOOP;
   END;
   $$ LANGUAGE plpgsql SECURITY DEFINER;
   ```
   Note: pg_cron scheduling (`cron.schedule(...)`) should be done manually in the Supabase SQL editor since it requires the pg_cron extension to be enabled. Add a comment in the migration with the cron command to run.

2. **Create `supabase/functions/delete-account/index.ts`:**
   - Follow Deno.serve pattern from RESEARCH.md
   - Accept `action` in JSON body: `'schedule'`, `'cancel'`, `'execute'`
   - Verify calling user from Authorization header using anon client
   - Use service_role client for admin operations
   - `schedule`: Set `deletion_scheduled_at` to now() + 7 days on profiles, also clean up avatar from storage (`storage.from('avatars').remove([userId + '/avatar.jpg'])` only on execute, not schedule)
   - `cancel`: Set `deletion_scheduled_at` to null
   - `execute`: Clean up storage avatar, then `auth.admin.deleteUser(userId)` (CASCADE handles data)
   - Return JSON responses with appropriate status codes
   - Handle CORS: Return 200 for OPTIONS requests with appropriate headers
   - Add CORS headers to all responses: `Access-Control-Allow-Origin: *`, `Access-Control-Allow-Headers: authorization, x-client-info, apikey, content-type`

3. **Add `deletionScheduledAt` to authStore** (`src/stores/authStore.ts`):
   - Add `deletionScheduledAt: string | null` to AuthState (default null)
   - Add `setDeletionScheduledAt: (date: string | null) => void` to AuthActions
   - Include in clearAuth reset

4. **Create `src/features/settings/hooks/useDeleteAccount.ts`:**
   - Export `useDeleteAccount()` returning `{ scheduleDelete, cancelDelete, deletionScheduledAt, isDeleting }`
   - `scheduleDelete(password: string)`:
     a. Verify password via `supabase.auth.signInWithPassword({ email: user.email, password })`
     b. If error, throw "Incorrect password"
     c. Call `supabase.functions.invoke('delete-account', { body: { action: 'schedule' } })`
     d. Parse response for `deletion_date`, store in authStore via `setDeletionScheduledAt`
     e. Call `supabase.auth.signOut()` and `clearAuth()` to sign user out after scheduling
   - `cancelDelete()`:
     a. Call `supabase.functions.invoke('delete-account', { body: { action: 'cancel' } })`
     b. Set `deletionScheduledAt` to null in authStore
   - On hook mount, check `profiles.deletion_scheduled_at` from Supabase and sync to store (for app restarts)
