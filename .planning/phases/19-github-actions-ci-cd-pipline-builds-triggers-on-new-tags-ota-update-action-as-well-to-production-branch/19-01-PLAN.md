<task type="auto">
  <name>Task 2: Build and OTA workflow files</name>
  <files>.github/workflows/build-ios.yml, .github/workflows/ota-update.yml</files>
  <action>
1. Create `.github/workflows/build-ios.yml`:
```yaml
name: iOS Build & Submit
on:
  push:
    tags:
      - 'v*'

permissions:
  contents: write

jobs:
  build:
    name: EAS Build iOS
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v5
      - uses: actions/setup-node@v6
        with:
          node-version: 22
          cache: npm
      - uses: expo/expo-github-action@v8
        with:
          eas-version: latest
          token: ${{ secrets.EXPO_TOKEN }}
      - run: npm ci
      - name: Build & Submit to TestFlight
        run: eas build --platform ios --profile production --non-interactive --auto-submit
      - name: Create GitHub Release
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
        run: |
          gh release create "${{ github.ref_name }}" \
            --repo="$GITHUB_REPOSITORY" \
            --title="${{ github.ref_name }}" \
            --generate-notes
```

Key details:
- `permissions: contents: write` required for `gh release create` (Pitfall 4 from research)
- Do NOT use `--no-wait` with `--auto-submit` (Pitfall 2 -- build must complete in-session for submit to trigger)
- `--non-interactive` prevents EAS CLI prompts in CI
- `--auto-submit` uses the `ascAppId` from eas.json submit config
- `--generate-notes` auto-creates changelog from commit messages since last tag
- iOS only per user decision (Android deferred)

2. Create `.github/workflows/ota-update.yml`:
```yaml
name: OTA Update
on:
  push:
    tags:
      - 'ota/*'

jobs:
  update:
    name: EAS Update Production
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v5
      - uses: actions/setup-node@v6
        with:
          node-version: 22
          cache: npm
      - uses: expo/expo-github-action@v8
        with:
          eas-version: latest
          token: ${{ secrets.EXPO_TOKEN }}
      - run: npm ci
      - name: Push OTA Update
        run: eas update --channel production --message "OTA ${{ github.ref_name }}" --non-interactive
```

Key details:
- `ota/*` tag pattern is mutually exclusive with `v*` (Pitfall 3 -- no overlap)
- Pushes JS bundle to `production` channel matching eas.json production profile
- Message includes the tag name for traceability on EAS dashboard
- No `permissions` block needed (no GitHub Release)

Tag patterns: `v*` matches `v1.0.0`, `v2.0.0-beta`, etc. `ota/*` matches `ota/v1.0.1`, `ota/hotfix-1`, etc. These are mutually exclusive.
