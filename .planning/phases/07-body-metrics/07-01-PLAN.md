    - Empty measurements array returns empty chart data

  <action>
    1. Create Supabase migration `supabase/migrations/20260315000000_create_body_measurements.sql`:
       - Table: public.body_measurements with columns: id (UUID PK), user_id (UUID FK to auth.users ON DELETE CASCADE), chest (NUMERIC(5,1) nullable), chest_unit (TEXT CHECK IN ('in','cm')), waist (NUMERIC(5,1) nullable), waist_unit (TEXT CHECK), hips (NUMERIC(5,1) nullable), hips_unit (TEXT CHECK), body_fat_pct (NUMERIC(4,1) nullable), measured_at (DATE NOT NULL DEFAULT CURRENT_DATE), created_at (TIMESTAMPTZ NOT NULL DEFAULT now())
       - CHECK constraint: at least one of chest, waist, hips, body_fat_pct is NOT NULL
       - Enable RLS, create policy "Users can manage own body measurements" FOR ALL USING (auth.uid() = user_id) WITH CHECK (auth.uid() = user_id)
       - Index on (user_id, measured_at DESC)
       - Allow multiple entries per day (no unique constraint on date)

    2. Create `src/features/body-metrics/types.ts`:
       ```typescript
       export interface BodyMeasurement {
         id: string;
         user_id: string;
         chest: number | null;
         chest_unit: 'in' | 'cm' | null;
         waist: number | null;
         waist_unit: 'in' | 'cm' | null;
         hips: number | null;
         hips_unit: 'in' | 'cm' | null;
         body_fat_pct: number | null;
         measured_at: string;
         created_at: string;
       }

       export type CircumferenceMetric = 'chest' | 'waist' | 'hips';
       export type BodyMetric = CircumferenceMetric | 'body_fat_pct';
       export type CircumferenceUnit = 'in' | 'cm';
       ```

    3. Create `src/features/body-metrics/utils/unitConversion.ts` with inchesToCm, cmToInches, convertMeasurement functions. Round to 1 decimal place.

    4. Create `src/stores/bodyMeasurementStore.ts` following exact same pattern as bodyweightStore (Phase 6) or historyStore (Phase 5):
       - Zustand + MMKV persistence with createMMKV({ id: 'body-measurement-storage' })
       - State: measurements[], isLoading, lastFetched
       - Actions: setMeasurements, addMeasurement, updateMeasurement, removeMeasurement, setLoading

    5. Create `src/features/body-metrics/hooks/useBodyMeasurements.ts`:
       - fetchMeasurements(): fetch from Supabase ordered by measured_at DESC, update store
       - addMeasurement(data): insert to Supabase, add to store (optimistic), return new entry
       - updateMeasurement(id, updates): update in Supabase, update store
       - deleteMeasurement(id): delete from Supabase, remove from store
       - Uses supabase.from('body_measurements' as any) pattern consistent with project

    6. Create `src/features/body-metrics/hooks/useBodyMetricsChartData.ts`:
       - Takes metric ('chest' | 'waist' | 'hips' | 'body_fat_pct') and displayUnit ('in' | 'cm')
       - Filters measurements where metric value is not null
       - For circumference metrics, converts all values to displayUnit using convertMeasurement
       - For body_fat_pct, ignores displayUnit (percentage is unitless)
       - Returns sorted array of { date: number (timestamp), value: number }
       - Uses useMemo for performance

    7. Write tests:
       - tests/body-metrics/unit-conversion.test.ts: Test inchesToCm, cmToInches, convertMeasurement with various inputs including same-unit (no-op) and cross-unit
       - tests/body-metrics/chart-data.test.ts: Test chart data transformation with mixed units, null filtering, sorting, empty input. Mock the store.
  </action>
