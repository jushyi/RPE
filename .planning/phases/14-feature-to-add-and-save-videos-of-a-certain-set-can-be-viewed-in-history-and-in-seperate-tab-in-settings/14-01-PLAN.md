  <action>
1. Create `videoUploadQueue.ts`:
   - Separate MMKV instance: `createMMKV({ id: 'video-upload-queue' })`
   - `enqueueVideoUpload(item: VideoUploadItem)`: Add to queue in MMKV
   - `getVideoQueue(): VideoUploadItem[]`: Read queue from MMKV
   - `removeFromQueue(setLogId: string)`: Remove item after successful upload
   - `flushVideoQueue()`: Check NetInfo connectivity, iterate queue, call uploadSetVideo for each, update set_logs.video_url via Supabase, remove successful items, keep failed items
   - Use `expo-file-system.copyAsync` to copy video from cache to documentDirectory immediately on enqueue (prevents stale URIs after app restart per Pitfall 6 in RESEARCH.md)

2. Create `thumbnailCache.ts`:
   - Uses MMKV to store thumbnail URI keyed by setLogId
   - `getCachedThumbnail(setLogId: string): string | null`
   - `cacheThumbnail(setLogId: string, uri: string): void`
   - `invalidateThumbnail(setLogId: string): void`
   - `generateAndCacheThumbnail(setLogId: string, videoUri: string): Promise<string>`: Uses expo-video-thumbnails `getThumbnailAsync(videoUri, { time: 500, quality: 0.5 })`, caches result

3. Create `useVideoCapture.ts` hook:
   - `captureOrPickVideo()`: Shows Alert with "Record Video" / "Choose from Gallery" / "Cancel" options (follows ProfilePhotoPicker pattern)
   - "Record Video": Requests camera permission, calls `ImagePicker.launchCameraAsync({ mediaTypes: ['videos'], videoMaxDuration: 120, videoQuality: ImagePicker.UIImagePickerControllerQualityType.Medium, allowsEditing: true })`
   - "Choose from Gallery": Requests media library permission, calls `ImagePicker.launchImageLibraryAsync({ mediaTypes: ['videos'], videoMaxDuration: 120, allowsEditing: true })`
   - Returns `{ localUri: string; thumbnailUri: string } | null`
   - After capture/pick, generates thumbnail via `generateAndCacheThumbnail`
   - Handles permission denial with user-friendly Alert

4. Create `useVideoUpload.ts` hook:
   - `uploadVideo(setLogId: string, localUri: string)`: Reads file as base64 via FileSystem.readAsStringAsync, converts with `decode` from base64-arraybuffer, uploads to `supabase.storage.from('set-videos').upload('{userId}/{setLogId}.mp4', decoded, { contentType: dynamicType, upsert: true })`
   - Determine contentType from file extension (.mov -> 'video/quicktime', .mp4 -> 'video/mp4') per Pitfall 1 in RESEARCH.md
   - Returns public URL via `getPublicUrl`
   - `deleteVideo(setLogId: string)`: Removes from storage and nulls video_url on set_logs

5. Create `VideoThumbnail.tsx`:
   - Props: `{ thumbnailUri: string; onPress: () => void; size?: number }`
   - Renders Image with play-circle Ionicons overlay
   - Pressable wrapper for tap handler
   - Dark theme styling consistent with app (colors from theme.ts)

6. Create `VideoCaptureButton.tsx`:
   - Props: `{ onVideoAttached: (localUri: string, thumbnailUri: string) => void; setLogId: string; hasVideo?: boolean }`
   - Renders Ionicons `videocam-outline` (or `videocam` if hasVideo) as a small Pressable button
   - On press, calls useVideoCapture.captureOrPickVideo(), then calls onVideoAttached with result
   - Shows ActivityIndicator while capturing
   - No emojis in UI (per CLAUDE.md)

7. Create `tests/videos/videoUploadQueue.test.ts`:
   - Mock MMKV with a simple in-memory map
   - Mock expo-file-system.copyAsync as a no-op
   - Test: `enqueueVideoUpload` adds item to queue, `getVideoQueue` returns it
   - Test: `removeFromQueue` removes the correct item by setLogId
   - Test: enqueuing multiple items returns all in `getVideoQueue`
   - Test: `removeFromQueue` with non-existent setLogId does not throw
   - Test: queue persists across calls (MMKV mock retains state within test)

8. Create `tests/videos/thumbnailCache.test.ts`:
   - Mock MMKV with a simple in-memory map
   - Test: `cacheThumbnail` stores URI, `getCachedThumbnail` retrieves it
   - Test: `getCachedThumbnail` returns null for unknown setLogId
   - Test: `invalidateThumbnail` removes cached entry
   - Test: overwriting a cached thumbnail replaces the old value
