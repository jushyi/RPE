    1. Create auth types (`src/features/auth/types.ts`):
       - `SignUpParams`: { email: string, password: string, displayName: string, photoUri?: string }
       - `SignInParams`: { email: string, password: string }
       - `AuthState`: { isAuthenticated: boolean, isLoading: boolean, user: User | null }

    2. Create `useAuth` hook (`src/features/auth/hooks/useAuth.ts`):
       - Listen to `supabase.auth.onAuthStateChange` in useEffect
       - On SIGNED_IN: update Zustand authStore (setAuthenticated, userId)
       - On SIGNED_OUT: clear Zustand authStore
       - `signUp({ email, password, displayName, photoUri })`:
         - Call `supabase.auth.signUp()` with email, password, and `options.data.display_name`
         - If photoUri provided, upload in background (non-blocking) per RESEARCH.md pattern
         - Photo upload: fetch blob from URI, upload to `avatars/{userId}/avatar.jpg` in Supabase Storage, update profiles.avatar_url on success
         - If upload fails, log warning but do NOT block signup flow
       - `signIn({ email, password })`: Call `supabase.auth.signInWithPassword()`
       - `signOut()`: Call `supabase.auth.signOut()`, clear authStore
       - Return: { signUp, signIn, signOut, isAuthenticated, isLoading, user }
       - IMPORTANT: Check network connectivity before signUp (sign-up requires internet per user decision). If offline, throw descriptive error.

    3. Create `useSession` hook (`src/features/auth/hooks/useSession.ts`):
       - On mount, check if Supabase has an existing session via `supabase.auth.getSession()`
       - BUT: check network status first. If offline, skip getSession() (it triggers token refresh that can fail and clear session). Instead, read auth state from Zustand/MMKV store.
       - Return: { isLoading: boolean, session: Session | null }

    **CHECKPOINT: Run `npx tsc --noEmit` after hooks to verify types before building components.**

    4. Create reusable UI components:
       - `src/components/ui/Button.tsx`: Dark/bold styled pressable with NativeWind. Props: title, onPress, variant ('primary' | 'secondary' | 'ghost'), loading (shows spinner), disabled. Primary uses accent blue bg, secondary uses surface bg with border.
       - `src/components/ui/Input.tsx`: Dark-themed TextInput wrapper. Props: label, error, placeholder, secureTextEntry, value, onChangeText. Dark surface bg, light text, red error border + message.

    5. Create `AuthForm` component (`src/features/auth/components/AuthForm.tsx`):
       - LOCKED DECISION: Single screen with toggle between sign-in and sign-up
       - Use react-hook-form + zod for validation
       - Sign-in mode: email + password fields + "Sign In" button + toggle link "Don't have an account? Sign up"
       - Sign-up mode: email + password + display name + ProfilePhotoPicker + "Create Account" button + toggle link "Already have an account? Sign in"
       - Zod schemas: email (valid email), password (min 6 chars), displayName (min 2 chars, sign-up only)
       - Show form-level error messages (e.g., "Invalid credentials", "Email already taken")
       - When offline in sign-up mode: show friendly message "Connect to the internet to create your account" with manual retry button (per user decision)
       - Dark/bold aesthetic: dark background, clean white inputs on surface color, accent blue buttons

    6. Create `ProfilePhotoPicker` (`src/features/auth/components/ProfilePhotoPicker.tsx`):
       - LOCKED DECISION: One default avatar available, plus camera/gallery picker
       - Display circular avatar preview (default avatar initially -- use a simple colored circle with initials or generic user icon)
       - Tap to show action sheet: "Take Photo", "Choose from Gallery", "Use Default"
       - Use expo-image-picker for camera (`launchCameraAsync`) and gallery (`launchImageLibraryAsync`)
       - Request permissions with friendly messaging if denied
       - Return selected URI via callback prop `onPhotoSelected(uri: string | null)`
