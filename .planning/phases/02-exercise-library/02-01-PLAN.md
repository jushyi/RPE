  <action>
    **1. Create Supabase migration** (`supabase/migrations/20260310000000_create_exercises.sql`):
    - CREATE TABLE public.exercises with columns: id (UUID PK default gen_random_uuid()), user_id (UUID nullable, references auth.users(id) ON DELETE CASCADE), name (TEXT NOT NULL), muscle_group (TEXT NOT NULL), equipment (TEXT NOT NULL), notes (TEXT nullable), created_at (TIMESTAMPTZ NOT NULL DEFAULT now()), updated_at (TIMESTAMPTZ NOT NULL DEFAULT now())
    - ENABLE ROW LEVEL SECURITY
    - Two separate SELECT policies (Postgres ORs them automatically):
      - "Anyone can read global exercises" USING (user_id IS NULL)
      - "Users can read own exercises" USING (auth.uid() = user_id)
    - INSERT policy: WITH CHECK (auth.uid() = user_id) — users can only insert with their own user_id
    - UPDATE policy: USING (auth.uid() = user_id) WITH CHECK (auth.uid() = user_id)
    - DELETE policy: USING (auth.uid() = user_id)
    - Indexes: idx_exercises_user_id ON exercises(user_id), idx_exercises_muscle_group ON exercises(muscle_group)
    - INSERT ~35 seed exercises (user_id = NULL) covering: Chest (Bench Press, Incline Bench Press, Decline Bench Press, Dumbbell Flyes, Cable Crossover), Back/Lats (Barbell Row, Deadlift, Lat Pulldown, Seated Cable Row, Pull-Up), Shoulders/Delts (Overhead Press, Lateral Raise, Face Pull, Arnold Press, Reverse Flyes), Biceps (Barbell Curl, Dumbbell Curl, Hammer Curl, Preacher Curl), Triceps (Triceps Pushdown, Overhead Triceps Extension, Skull Crushers, Close-Grip Bench Press), Quads (Squat, Leg Press, Leg Extension, Bulgarian Split Squat), Hamstrings (Romanian Deadlift, Leg Curl), Glutes (Hip Thrust, Glute Bridge), Calves (Calf Raise), Core (Plank, Hanging Leg Raise, Cable Crunch), Traps (Barbell Shrug, Dumbbell Shrug)
    - Use detailed muscle group names matching the MuscleGroup type: Chest, Lats, Delts, Biceps, Triceps, Forearms, Quads, Hamstrings, Glutes, Calves, Core, Traps

    **2. Create exercise type definitions** (`src/features/exercises/types.ts`):
    - Export MuscleGroup union type: 'Chest' | 'Lats' | 'Delts' | 'Biceps' | 'Triceps' | 'Forearms' | 'Quads' | 'Hamstrings' | 'Glutes' | 'Calves' | 'Core' | 'Traps'
    - Export Equipment union type: 'Barbell' | 'Dumbbell' | 'Cable' | 'Machine' | 'Bodyweight' | 'Kettlebell' | 'Band' | 'Other'
    - Export Exercise interface matching the migration schema
    - Export isCustomExercise helper (user_id !== null)

    **3. Create constants** (`src/features/exercises/constants/muscleGroups.ts` and `equipmentTypes.ts`):
    - muscleGroups.ts: MUSCLE_GROUP_COLORS record mapping each MuscleGroup to a distinct hex color (use bold, high-contrast colors that pop on dark backgrounds — red, blue, amber, violet, pink, orange, green, teal, rose, cyan, yellow, indigo)
    - muscleGroups.ts: MUSCLE_GROUPS array derived from the color map keys
    - equipmentTypes.ts: EQUIPMENT_TYPES array of all Equipment values

    **4. Create Zustand exercise store** (`src/stores/exerciseStore.ts`):
    - Follow exact pattern from authStore.ts
    - Use createMMKV({ id: 'exercise-storage' }) — MUST use named instance to avoid colliding with auth store's default instance
    - State: exercises (Exercise[]), isLoading (boolean), lastFetched (number | null)
    - Actions: setExercises, addExercise, updateExercise, removeExercise, setLoading
    - Persist with createJSONStorage wrapping the named MMKV instance

    **5. Extend database types** (`src/lib/supabase/types/database.ts`):
    - Add Exercise interface (import from features/exercises/types.ts or define inline matching migration)
    - Add exercises table to Database.public.Tables with Row, Insert, Update shapes matching existing pattern

    **6. Create useExercises hook** (`src/features/exercises/hooks/useExercises.ts`):
    - fetchExercises: query supabase.from('exercises').select('*').order('name'), store result in exerciseStore. Only fetch if not already loaded (check lastFetched).
    - createExercise: insert with user_id from authStore, add to store optimistically
    - updateExercise: update in supabase, update in store
    - deleteExercise: delete from supabase, remove from store
    - Return { exercises, isLoading, fetchExercises, createExercise, updateExercise, deleteExercise }

    **7. Create test files:**
    - `tests/exercises/exercise-store.test.ts`: Test exerciseStore actions (setExercises, addExercise, updateExercise, removeExercise) in isolation using Zustand's vanilla store pattern
    - `tests/exercises/exercise-library.test.ts`: Test filtering logic — given a set of exercises, verify that muscle group filter, equipment filter, and search query narrow results correctly. Test the useMemo filter logic as a pure function extracted for testability.
