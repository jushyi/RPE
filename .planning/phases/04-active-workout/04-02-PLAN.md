    1. Run `npx expo install react-native-pager-view` to add the dependency.

    2. Create `app/(app)/workout/_layout.tsx`: Stack layout with no header (headerShown: false) for the workout group. This is a modal-like full-screen experience.

    3. Create `src/features/workout/hooks/useWorkoutSession.ts`:
       - Wraps useWorkoutStore with session lifecycle convenience methods
       - `startFromPlan(planDay)`: reads userId from authStore, calls workoutStore.startPlanSession, navigates to /workout
       - `startFreestyle()`: reads userId from authStore, calls workoutStore.startFreestyleSession, navigates to /workout
       - `logCurrentSet(weight, reps, unit)`: creates SetLog with crypto.randomUUID(), auto-increments set_number based on current exercise's logged_sets.length + 1, calls workoutStore.logSet
       - `finishWorkout()`: calls workoutStore.finishSession(), navigates to /workout/summary with session data
       - `endEarly()`: shows Alert.alert confirmation "You have X exercises remaining. End anyway?", if confirmed calls finishWorkout()
       - `addFreestyleExercise(exercise)`: creates SessionExercise from Exercise, calls workoutStore.addExercise, navigates PagerView to new exercise
       - Returns: session, currentExercise, exerciseCount, currentIndex, logCurrentSet, finishWorkout, endEarly, addFreestyleExercise

    4. Create `src/features/workout/components/WorkoutHeader.tsx`:
       - Displays: current exercise name (large, bold), set progress "Set X of Y", end workout button (top right)
       - Uses StyleSheet.create, dark background, white text
       - End workout button calls endEarly() if exercises remain, finishWorkout() if all done

    5. Create `src/features/workout/components/FreestyleExercisePicker.tsx`:
       - Reuses the exercise library search/filter pattern from Phase 2
       - Renders in a BottomSheetModal (same pattern as exercise CRUD)
       - On exercise select: creates SessionExercise with blank target_sets, null weight_progression, calls addFreestyleExercise
       - Present via ref from a "+" FAB button on the workout screen

    6. Create `app/(app)/workout/index.tsx`:
       - Reads activeSession from workoutStore. If null and no route params, redirect back.
       - Route params: `plan_day_id` (optional, string) — if provided, load plan day from planStore and call startFromPlan
       - If no plan_day_id, check if activeSession exists (resume), otherwise this is a freestyle start
       - Renders WorkoutHeader + ExercisePager (Task 2) + FAB for freestyle picker
       - Full screen, dark background, no status bar overlap (SafeAreaView)
       - KeyboardAvoidingView wrapping with behavior="padding" (iOS) / "height" (Android)

    7. Create `app/(app)/workout/summary.tsx`:
       - Shell screen that receives completed session data
       - Displays: "Workout Complete" heading, placeholder for stats (filled in Plan 04)
       - "Done" button navigates back to tabs
       - Uses StyleSheet.create, dark theme
