1. **src/stores/coachingStore.ts**: Create Zustand + MMKV store following exact pattern from planStore.ts:
   - State: `relationships: CoachingRelationship[]`, `trainees: TraineeProfile[]`, `coaches: TraineeProfile[]`, `isLoading: boolean`
   - Actions: `setRelationships`, `addRelationship`, `removeRelationship`, `setTrainees`, `setCoaches`, `setLoading`
   - Named MMKV instance: `coaching-storage`

2. **src/features/coaching/hooks/useCoaching.ts**: Export `useCoaching()` hook with:

   a. `fetchRelationships()`: Query coaching_relationships where coach_id = userId OR trainee_id = userId. For each relationship, fetch the other party's profile (display_name, avatar_url) from profiles. Separate into trainees (where current user is coach) and coaches (where current user is trainee). Store all in coachingStore.

   b. `generateCode()`: Call `generateInviteCode()` utility, insert into invite_codes table with coach_id = userId and expires_at = now + 24h. Return the code string. If code collides (unique constraint), retry once with a new code.

   c. `redeemCode(code: string)`: Query invite_codes where code matches, not redeemed, not expired. If found, UPDATE redeemed_by = userId, then INSERT into coaching_relationships (coach_id from invite, trainee_id = userId). Return the new relationship. Throw if code invalid/expired/already-redeemed.

   d. `disconnect(relationshipId: string)`: DELETE from coaching_relationships by id. Remove from store.

   e. `getActiveInviteCode()`: Query invite_codes where coach_id = userId, not redeemed, not expired. Return the most recent one or null. (So coach can see their pending code.)

   All Supabase calls use `as any` pattern per project convention for .from() calls.
