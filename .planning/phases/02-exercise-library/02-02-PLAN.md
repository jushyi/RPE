    **3. Create ExerciseBottomSheet** (`src/features/exercises/components/ExerciseBottomSheet.tsx`):
    - Use BottomSheetModal from @gorhom/bottom-sheet (not BottomSheet — Modal variant works with the Provider and supports present/dismiss)
    - Accept ref (forwardRef pattern with BottomSheetModal ref type) and optional exerciseToEdit (Exercise | null) prop for edit mode
    - Accept onSave callback prop
    - Snap points: ['70%'] (enough room for form + keyboard)
    - Use BottomSheetTextInput (not regular TextInput) for inputs inside the sheet — critical for Android keyboard behavior
    - Form fields using react-hook-form + zod validation:
      - name: required, min 1 char (BottomSheetTextInput)
      - muscle_group: required, picker — render as a scrollable list of Pressable chips (reuse muscle group chip styling from ExerciseFilterBar). Active selection highlighted.
      - equipment: required, picker — same chip-based selection pattern
      - notes: optional freeform (BottomSheetTextInput, multiline)
    - Duplicate name warning: On submit, check if any exercise in the store has the same name (case-insensitive). If match found, show inline warning text below the name field ("An exercise with this name already exists") but do NOT block submission.
    - Submit button: "Add Exercise" in create mode, "Save Changes" in edit mode
    - On submit: call createExercise or updateExercise from useExercises hook, then dismiss the sheet
    - When exerciseToEdit changes to non-null, pre-fill form fields with exercise data via reset()
    - Dark theme styling: BottomSheetModal backgroundStyle with colors.surface, handleIndicatorStyle with colors.textMuted
    - BottomSheetModal props: keyboardBehavior="interactive", android_keyboardInputMode="adjustResize"

    **4. Wire bottom sheet into exercises screen** (`app/(app)/(tabs)/exercises.tsx`):
    - Add a ref for BottomSheetModal (useRef)
    - Add local state: exerciseToEdit (Exercise | null)
    - FAB / "Add Exercise" button: set exerciseToEdit to null, present bottom sheet
    - ExerciseListItem onLongPress: show Alert.alert with title exercise.name and buttons:
      - "Edit" (only for custom exercises): set exerciseToEdit, present bottom sheet
      - "Delete" (only for custom exercises): confirm via Alert.alert, then call deleteExercise
      - "Cancel"
    - For global/seed exercises, long-press does nothing or shows info-only alert ("This is a built-in exercise")
    - On bottom sheet onSave callback: refresh is automatic since createExercise/updateExercise already update the Zustand store
    - FAB styling: absolute positioned at bottom-right, circular, accent color background, "+" text, elevation/shadow

    **5. Create CRUD test** (`tests/exercises/exercise-crud.test.ts`):
    - Test createExercise adds to store
    - Test updateExercise modifies correct exercise in store
    - Test deleteExercise removes from store
    - Test duplicate name detection logic (case-insensitive match)
    - Mock Supabase client for insert/update/delete operations
