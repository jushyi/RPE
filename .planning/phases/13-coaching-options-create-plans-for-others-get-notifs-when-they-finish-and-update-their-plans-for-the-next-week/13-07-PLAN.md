Three changes to InviteCodeModal.tsx, all in the same file:

**1. Fade overlay + slide content (instead of sliding everything together):**
- Change `animationType="slide"` to `animationType="fade"` on the Modal component. This makes the dark overlay fade in.
- Wrap the inner modal content (`View style={s.modal}`) with a Reanimated Animated.View that slides up from below using `useAnimatedStyle` with `translateY`. On `visible=true`, animate translateY from 300 to 0 using `withTiming` (duration ~300ms). Use a `useEffect` watching `visible` to trigger the animation.
- Import `Animated, useSharedValue, useAnimatedStyle, withTiming` from 'react-native-reanimated'.

**2. Tap outside to dismiss:**
- Replace the outer `<View style={s.overlay}>` with `<Pressable style={s.overlay} onPress={onClose}>`.
- Wrap the inner modal content in a nested `<Pressable onPress={(e) => e.stopPropagation()}>` (or use `onStartShouldSetResponder={() => true}` on the inner View) to prevent taps on the sheet content from bubbling up and dismissing.
- Simpler alternative: Use a Pressable for the overlay, and wrap the sheet content View with a Pressable that has `onPress={undefined}` -- the key is that tapping the sheet itself should NOT dismiss, only tapping the dark area above.

**3. Keyboard avoidance for Enter Code tab:**
- Import `KeyboardAvoidingView` and `Platform` from 'react-native'.
- Wrap the modal content (the sheet, not the overlay) in `<KeyboardAvoidingView behavior={Platform.OS === 'ios' ? 'padding' : 'height'} style={{ width: '100%' }}>`.
- This ensures when the user taps the 6-character code input, the keyboard pushes the sheet up so the input remains visible.

Keep all existing functionality (tabs, generate, enter, copy, redeem) unchanged.
