  <action>
Create the Edge Function following the exact pattern from delete-account/index.ts. The function:

1. **CORS preflight** handling (OPTIONS returns 'ok').

2. **Auth verification**: Extract Authorization header. Create a user client to verify the calling user (same pattern as delete-account). This ensures only authenticated users can trigger push notifications.

3. **Service role client**: Create admin client with SUPABASE_SERVICE_ROLE_KEY for reading push_tokens table (since users can't read other users' tokens via RLS).

4. **Request body**: Parse JSON with fields:
   - `recipient_ids: string[]` - user IDs to notify
   - `title: string` - notification title
   - `body: string` - notification body text
   - `data?: Record<string, any>` - optional notification data payload

5. **Fetch push tokens**: Using admin client, query push_tokens table for all recipient_ids. If no tokens found, return `{ sent: 0, reason: 'no_tokens' }` with 200 status.

6. **Send via Expo Push API**: POST to `https://exp.host/--/api/v2/push/send` with:
   - Content-Type: application/json
   - Authorization: Bearer ${Deno.env.get('EXPO_ACCESS_TOKEN')} (optional, omit header if env var not set)
   - Body: array of message objects, each with { to: token, sound: 'default', title, body, data }

7. **Return result**: `{ sent: N, result }` with Expo API response.

8. **Error handling**: try/catch returning 500 with `{ error: 'Internal server error' }`.

Keep the function generic -- it does not know about coaching concepts. It just sends push notifications to a list of user IDs.
