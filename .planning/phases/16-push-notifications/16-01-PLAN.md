    Create notificationStore following the exact Zustand + MMKV pattern from existing stores (historyStore, planStore):

    1. Create MMKV instance with id 'notification-storage'
    2. State shape:
       - notifications: NotificationRecord[] (sorted by created_at DESC)
       - unreadCount: number
       - isLoading: boolean
       - lastFetched: number | null

    3. Actions:
       - fetchNotifications(): Fetch from Supabase notifications table (user's own, ordered by created_at DESC, limit 50). Prune records older than 30 days client-side (DELETE then re-fetch). Update unreadCount.
       - refreshUnreadCount(): SELECT count(*) from notifications WHERE read = false for current user. Update unreadCount.
       - markAsRead(id: string): UPDATE notifications SET read = true WHERE id = id. Update local state and decrement unreadCount.
       - markAllRead(): UPDATE notifications SET read = false (WHERE user_id = current user AND read = false). Set all local notifications to read, unreadCount to 0. Also call Notifications.setBadgeCountAsync(0).
       - addLocalNotification(record: Omit<NotificationRecord, 'id' | 'created_at'>): INSERT into Supabase table (fire-and-forget for local alarm/nudge persistence). Add to local state.

    4. Persist with MMKV using the same custom storage adapter pattern as other stores.

    Use `as any` for Supabase `.from('notifications')` calls since the table isn't in generated types yet (matching existing pattern from Phase 13).
