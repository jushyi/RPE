<task type="auto">
  <name>Task 1: Create interactive device verification script</name>
  <files>scripts/verify-device.js</files>
  <action>
Create a Node.js CLI script at scripts/verify-device.js that walks the user through each physical device test case interactively. Use Node.js readline (not shell script) for Windows compatibility per user's platform.

The script should:

1. Display a header: "RPE - Physical Device Verification" with the current date/time.

2. Define these 5 test cases (from phase success criteria, skipping progress photos per user decision):

   TF-01: App installs and launches from TestFlight
   - Instructions: "Open TestFlight on your iPhone. Accept the RPE invite. Install the app. Launch it and confirm the login screen appears."

   ALM-01: Alarm fires with sound and vibration (iOS)
   - Instructions: "Create or edit a plan with a training day. Set the alarm time to 2 minutes from now. Lock the phone and wait. Verify the alarm fires with sound and vibration and requires dismissal."

   OFF-01: Offline workout logging and sync
   - Instructions: "Enable airplane mode on the iPhone. Start a workout (plan-based or freestyle). Log at least 3 sets across 2 exercises. Finish the workout. Disable airplane mode. Wait 10 seconds. Open History tab and verify the session appears with all logged data."

   RLS-01: RLS cross-user data isolation
   - Instructions: "Log out of the current account. Create a NEW account with a different email (e.g., testuser2@example.com). After login, verify: (1) Dashboard shows no workout history, (2) History tab is empty, (3) Plans tab shows no plans, (4) Exercise library shows only default exercises (no custom exercises from first account). Log out and log back in to the original account to verify its data is still intact."

   HIST-01: Plan-edit does not alter logged sessions
   - Instructions: "Note the exercises and set details in a previously logged session (from History tab). Go to Plans tab and edit that plan -- change an exercise name or remove an exercise. Go back to History and verify the old session still shows the ORIGINAL exercises and data unchanged."

3. For each test, display the test ID, name, and instructions. Prompt for result: pass / fail / skip. Prompt for optional notes.

4. After all tests, display a summary table showing test ID, name, result. Count passes, fails, skips.

5. Write results to `scripts/verification-results.json` with structure:
```json
{
  "date": "ISO timestamp",
  "device": "Physical iPhone (TestFlight)",
  "app_version": "1.0.0",
  "results": [
    { "id": "TF-01", "name": "...", "result": "pass|fail|skip", "notes": "..." }
  ],
  "summary": { "pass": N, "fail": N, "skip": N, "total": 5 }
}
```

6. If any test fails, print a clear message: "DISTRIBUTION BLOCKED -- {N} test(s) failed. Fix issues and re-run verification."
   If all pass (with skips OK): "VERIFICATION PASSED -- App is ready for distribution to friends."

Use console colors (ANSI escape codes) for pass (green), fail (red), skip (yellow) in the summary. Keep the script self-contained with no external dependencies (no npm packages needed).

Do NOT use emojis in any output per project convention.
