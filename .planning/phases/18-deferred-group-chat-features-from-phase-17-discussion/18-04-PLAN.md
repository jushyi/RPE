  <files>src/features/social/hooks/useTypingIndicator.ts, src/features/social/hooks/useReadReceipts.ts, src/features/social/components/TypingIndicator.tsx, tests/chat/typingDebounce.test.ts</files>
  <behavior>
    - Typing debounce: calling onTextChange sets typing=true, after 2s of no calls sets typing=false
    - Typing debounce: rapid calls reset the 2s timer
    - Read receipts: markAsRead upserts last_read_message_id for current user
    - getMessageReadStatus returns correct status based on member receipts
  </behavior>
  <action>
    Write typing debounce test FIRST, then implement:

    1. `tests/chat/typingDebounce.test.ts`: Test the debounce logic as a pure function (extract debounce behavior from hook). Test that rapid calls reset timer, that typing becomes false after 2s idle.

    2. `useTypingIndicator(groupId)` hook:
       - Join the same `group-chat:${groupId}` channel with presence config (key: userId).
       - On 'presence' 'sync' event: extract users where typing===true (exclude self), map to display names.
       - `startTyping()`: call channel.track({ typing: true, user_name: displayName }). Start 2-second timeout. On timeout, track({ typing: false, user_name: displayName }).
       - `stopTyping()`: immediately track typing: false, clear timeout.
       - Listen to AppState: on background, call channel.untrack() to prevent ghost typing indicators (per Pitfall 5 in RESEARCH.md).
       - Cleanup: untrack + removeChannel on unmount.
       - Return { typingUsers: string[], startTyping }.

    3. `useReadReceipts(groupId)` hook:
       - `markAsRead(messageId)`: upsert into group_read_receipts with onConflict 'group_id,user_id'.
       - `fetchReceipts()`: load all group_read_receipts for groupId.
       - Auto-mark as read when chat is focused (call markAsRead with latest message ID).
       - Subscribe to postgres_changes on group_read_receipts for real-time read status updates.
       - Return { receipts, markAsRead }.

    4. `TypingIndicator` component:
       - Shows "[Name] is typing..." or "[Name1] and [Name2] are typing..." at the bottom of the chat, above MessageInput.
       - Subtle animation (opacity pulse or dots animation).
       - Hidden when typingUsers array is empty.
  </action>

  </verify>
