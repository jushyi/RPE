    Sync Queue:
    - enqueueSyncItem(item) appends to pending queue in MMKV
    - flushSyncQueue(supabase) processes all pending items; failed items remain in queue
    - Empty queue returns immediately
    - Offline check: skips flush if not connected

    Session Summary:
    - Computes duration from started_at to ended_at
    - Computes total volume as sum of (weight * reps) across all logged sets
    - Counts exercises with at least one logged set
    - Counts sets with is_pr = true
  </behavior>
  <action>
    1. Create `src/features/workout/hooks/useSyncQueue.ts` following RESEARCH.md Pattern 4:
       - Named MMKV instance `sync-queue`
       - `enqueueSyncItem(item: SyncItem)`: append to pending array in MMKV
       - `flushSyncQueue(supabase)`: check NetInfo.fetch() for connectivity, process each item with supabase.from(table)[operation](data), failed items stay in queue
       - `useSyncQueue()` hook: subscribes to NetInfo connection changes, auto-flushes when connectivity restored. Also flushes on session completion.
       - `enqueueCompletedSession(session: WorkoutSession)`: breaks a completed session into sync items: one insert for workout_sessions, one insert per session_exercise, one insert per set_log. Enqueues all items. Then triggers flush.

    2. Create `src/features/workout/components/SessionSummary.tsx`:
       - Takes a completed WorkoutSession as prop
       - Computes SessionSummary: duration (diff started_at to ended_at in minutes), total volume (sum weight*reps across all sets), exercises completed (count exercises with logged_sets.length > 0), PRs hit (count sets with is_pr true)
       - Displays as a stats card: 4 stat boxes in a 2x2 grid (Duration, Volume, Exercises, PRs)
       - Dark theme, bold numbers, secondary labels
       - StyleSheet.create, no emojis — use icon-like text or Ionicons for stats

    3. Create `src/features/workout/components/WeightTargetPrompt.tsx`:

       - If none, renders null
