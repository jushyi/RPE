  <name>Task 1: Database migration for coaching system</name>
  <files>supabase/migrations/20260317000001_create_coaching.sql</files>
  <action>
Create a single migration file containing ALL coaching schema. Use the exact schema from RESEARCH.md with these specifics:

1. **push_tokens table**: user_id (PK, FK auth.users ON DELETE CASCADE), token TEXT NOT NULL, platform TEXT CHECK ('ios','android'), updated_at TIMESTAMPTZ. RLS: users manage own token only.

2. **invite_codes table**: id UUID PK, coach_id FK auth.users, code TEXT UNIQUE NOT NULL, expires_at TIMESTAMPTZ NOT NULL, redeemed_by UUID nullable FK, created_at. RLS: coach manages own codes; anyone can SELECT unexpired unredeemed codes; anyone can UPDATE (redeem) unredeemed codes setting redeemed_by = auth.uid().

3. **coaching_relationships table**: id UUID PK, coach_id FK, trainee_id FK, created_at. UNIQUE(coach_id, trainee_id), CHECK(coach_id != trainee_id). RLS: either party can SELECT; trainee can INSERT (via code redemption); either party can DELETE.

4. **coach_notes table**: id UUID PK, plan_id FK workout_plans ON DELETE CASCADE, coach_id FK auth.users, note TEXT NOT NULL, created_at. RLS: coach can INSERT/SELECT own notes; trainee can SELECT notes on plans where user_id = auth.uid().

5. **is_coach_of() helper function**: SECURITY DEFINER, STABLE, checks coaching_relationships for auth.uid() as coach_id.

6. **ALTER workout_plans**: Add coach_id UUID nullable FK auth.users ON DELETE SET NULL. Add 4 new RLS policies:
   - Coaches can SELECT plans they created (coach_id = auth.uid())
   - Coaches can INSERT plans for trainees (coach_id = auth.uid() AND is_coach_of(user_id))
   - Coaches can UPDATE plans they created (coach_id = auth.uid())
   - Coaches can DELETE plans they created (coach_id = auth.uid())

7. **Coach read access to workout data**: Add SELECT policies on workout_sessions (is_coach_of(user_id)), session_exercises (JOIN through workout_sessions), set_logs (JOIN through session_exercises+workout_sessions). Also allow coaches to SELECT trainee profiles (is_coach_of(id)).

8. **Coach access to plan sub-tables**: Add SELECT/INSERT/UPDATE/DELETE policies on plan_days for plans where coach_id = auth.uid(). Same for plan_day_exercises (JOIN through plan_days + workout_plans).

Follow the existing migration naming convention (timestamp prefix). Use 20260317000001 as the timestamp.
