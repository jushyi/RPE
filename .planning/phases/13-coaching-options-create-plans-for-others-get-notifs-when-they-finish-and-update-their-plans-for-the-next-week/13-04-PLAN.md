  <name>Task 1: Coach plan hooks (CRUD + performance data + trainee history)</name>
  <files>src/features/coaching/hooks/useCoachPlans.ts, src/features/coaching/hooks/useTraineePerformance.ts, src/features/coaching/hooks/useTraineeHistory.ts, src/features/coaching/components/CoachNoteInput.tsx, src/features/coaching/components/InlinePerformance.tsx</files>
  <action>
1. **useCoachPlans.ts**: Export `useCoachPlans(traineeId: string)` hook:
   - `fetchTraineePlans()`: Query workout_plans where user_id = traineeId AND (coach_id = current userId OR coach_id IS NULL). Include plan_days and plan_day_exercises with exercises. This lets coach see both their coach-created plans AND the trainee's personal plans (personal plans are read-only for coach).
   - `createPlanForTrainee(name, days)`: Same logic as usePlans.createPlan but inserts with user_id = traineeId and coach_id = current userId. Does NOT schedule alarms (trainee sets own alarms per discretion decision).
   - `updateTraineePlan(planId, name, days, note?)`: Delete-and-reinsert approach (same pattern as existing plan edit). If note is provided, INSERT into coach_notes table (plan_id, coach_id, note). Only works on plans where coach_id = current userId.
   - `deleteTraineePlan(planId)`: DELETE workout_plans where id = planId AND coach_id = current userId.
   - All queries use `as any` Supabase pattern.

2. **useTraineePerformance.ts**: Export `useTraineePerformance(traineeId: string, exerciseIds: string[])` hook:
   - Fetches the trainee's most recent workout data for each exercise_id within the last 7 days.
   - Query: session_exercises with set_logs, joined through workout_sessions where user_id = traineeId and ended_at >= 7 days ago.
   - Returns a Map<exerciseId, { bestWeight: number, bestReps: number, totalSets: number, unit: string }>.
   - Auto-fetches when traineeId or exerciseIds change.
   - Returns empty map while loading.

3. **useTraineeHistory.ts**: Export `useTraineeHistory(traineeId: string)` hook:
   - `fetchSessions()`: Query workout_sessions where user_id = traineeId, ordered by started_at DESC. Include session_exercises with exercise name and set_logs for set detail.
   - Returns `{ sessions: WorkoutSession[], isLoading: boolean, fetchMore: () => void }`.
   - Paginate with limit 20, load more on scroll.
   - Uses coach RLS (is_coach_of) to access trainee's workout data -- no special auth needed beyond existing RLS policies from Plan 01.

4. **CoachNoteInput.tsx**: A collapsible text input component for coach notes:
   - Shows "Add note for trainee" label with Ionicons chatbox-outline icon.
   - TextInput with multiline, max 200 chars, placeholder "e.g., Great week, bumping your bench target".
   - Accepts `value` and `onChangeText` props.
   - Dark theme styling per project convention.

5. **InlinePerformance.tsx**: Display component showing trainee's last-week actuals:
   - Receives performanceData (from useTraineePerformance map) for a single exercise.
   - Renders: "Last week: {bestWeight}{unit} x {bestReps} ({totalSets} sets)" in a muted/secondary text style.
   - If no data for exercise, shows "No recent data" in dimmed text.
   - Designed to slot in next to plan target values in the exercise row.

After creating hooks, uncomment imports in tests/coaching/coachPlans.test.ts and fill in any testable pure-logic tests.
