# Phase 15.1: Reverse Bar Loading + Customizable Plate Inventory - Research

**Researched:** 2026-03-13
**Domain:** React Native UI enhancement (calculator feature extension)
**Confidence:** HIGH

## Summary

This phase adds two features to the existing PlateCalculator component: (1) a reverse mode where users tap +/- steppers to build plates onto a bar and see the total weight, and (2) a "My Plates" inventory section where users toggle which plate sizes they own, persisted per unit system in MMKV.

The existing codebase is well-suited for this. The `calculatePlates()` function already accepts an `availablePlates` parameter, so filtering by inventory is a one-line change. The `BarbellDiagram` component already renders from a `plates` array, so it works identically for both modes. The main work is UI: a segmented mode toggle, stepper rows for reverse mode, and a collapsible inventory section.

**Primary recommendation:** Extend `PlateCalculator.tsx` with mode state, add a new `PlateStepperList` component for reverse mode, add a `MyPlatesSection` collapsible component, and create a lightweight MMKV-backed `usePlateInventory` hook for persistence. No new libraries needed.

### Pattern 1: MMKV Direct Storage (No Zustand)
**What:** Lightweight MMKV hook without full Zustand store overhead
**When to use:** Simple key-value persistence that does not need cross-component reactivity or Supabase sync
**Why here:** Plate inventory is local-only, read/written within a single component tree. A full Zustand store is overkill.

```typescript
import { createMMKV } from 'react-native-mmkv';

const mmkv = createMMKV({ id: 'plate-inventory' });

const LB_KEY = 'inventory-lb';
const KG_KEY = 'inventory-kg';

export function usePlateInventory(unit: 'kg' | 'lbs') {
  const key = unit === 'kg' ? KG_KEY : LB_KEY;
  const allPlates = unit === 'kg' ? KG_PLATES : LB_PLATES;

  // Read from MMKV (synchronous)
  const stored = mmkv.getString(key);
  const initial: number[] = stored ? JSON.parse(stored) : [...allPlates];

  const [enabledPlates, setEnabledPlates] = useState<number[]>(initial);

  const toggle = (weight: number) => {
    setEnabledPlates(prev => {
      const next = prev.includes(weight)
        ? prev.filter(w => w !== weight)
        : [...prev, weight].sort((a, b) => b - a);
      mmkv.set(key, JSON.stringify(next));
      return next;
    });
  };

  // Re-read when unit changes
  useEffect(() => {
    const s = mmkv.getString(key);
    setEnabledPlates(s ? JSON.parse(s) : [...allPlates]);
  }, [unit]);

  return { enabledPlates, allPlates, toggle };
}
```

### Pattern 2: Reverse Mode Total Weight Calculation
**What:** Compute total barbell weight from per-side plate counts
**When to use:** Plates > Weight mode

```typescript
// plateCounts: Record<number, number> = { 45: 2, 25: 1 } means 2x45 + 1x25 per side
function calculateTotalWeight(
  plateCounts: Record<number, number>,
  barWeight: number
): number {
  const perSide = Object.entries(plateCounts).reduce(
    (sum, [weight, count]) => sum + Number(weight) * count, 0
  );
  return barWeight + perSide * 2;
}
```

## Common Pitfalls

### Pitfall 1: Stale Inventory When Switching Units
**What goes wrong:** User switches from lb to kg but sees lb inventory state
**Why it happens:** React state not re-reading from MMKV when unit prop changes
**How to avoid:** `useEffect` on unit change that re-reads MMKV for the new unit's key
**Warning signs:** Toggling a kg plate and switching to lb shows that plate toggled too

### Pitfall 2: Floating Point in Reverse Mode Total
**What goes wrong:** Adding 2x2.5 plates shows 49.99999 instead of 50
**Why it happens:** JavaScript floating point arithmetic
**How to avoid:** Round total weight display: `Math.round(total * 100) / 100` (same pattern used in existing `calculatePlates`)
**Warning signs:** Decimal weights appearing with many trailing digits

### Pitfall 3: Empty Enabled Plates Array
**What goes wrong:** User disables all plates, then weight-to-plates mode crashes or shows empty
**Why it happens:** `calculatePlates()` called with empty `availablePlates`
**How to avoid:** Show a message like "Enable at least one plate size in My Plates" when all disabled. Prevent last plate from being disabled, or handle gracefully.
**Warning signs:** Empty plates array passed to calculatePlates

### Pitfall 4: BarbellDiagram viewBox Overflow
**What goes wrong:** Too many plates overflow the SVG viewBox, causing clipping
**Why it happens:** Fixed TOTAL_WIDTH constant in current BarbellDiagram
**How to avoid:** Make TOTAL_WIDTH dynamic based on actual plate count (pass as prop or compute from plates)
**Warning signs:** Plates visually cut off at the right edge of diagram

### Pitfall 5: Remainder Warning Missing Plate Name
**What goes wrong:** Generic "cannot load exactly" warning without mentioning which plate is missing
**Why it happens:** Current warning only shows remainder amount, not which plate would solve it
**How to avoid:** Compare `LB_PLATES`/`KG_PLATES` full list against enabled plates to identify the smallest missing plate that would help
**Warning signs:** User sees remainder but doesn't know which plate to add to inventory

### Pitfall 6: Inventory Collapsed State Not Persisting
**What goes wrong:** "My Plates" section always starts expanded on app relaunch
**Why it happens:** Collapsed state tracked in React state only
**How to avoid:** Persist collapsed state in MMKV alongside inventory (or use "collapsed by default once configured" per user decision -- collapse when not all plates are enabled)
**Warning signs:** Section always expanded on fresh mount

### Stepper Row Component
```typescript
// PlateStepperList.tsx
interface PlateStepperRowProps {
  weight: number;
  count: number;
  unit: 'kg' | 'lbs';
  onIncrement: () => void;
  onDecrement: () => void;
}

function PlateStepperRow({ weight, count, unit, onIncrement, onDecrement }: PlateStepperRowProps) {
  const plateColors = unit === 'kg' ? PLATE_COLORS_KG : PLATE_COLORS_LB;
  const color = plateColors[weight] ?? '#666';
  const unitLabel = unit === 'kg' ? 'kg' : 'lb';

  return (
    <View style={s.stepperRow}>
      <View style={[s.plateChip, { backgroundColor: color }]}>
        <Text style={s.plateChipText}>{weight} {unitLabel}</Text>
      </View>
      <View style={s.stepperControls}>
        <Pressable onPress={onDecrement} disabled={count === 0} style={s.stepBtn}>
          <Ionicons name="remove-circle-outline" size={28} color={count === 0 ? colors.textMuted : colors.textPrimary} />
        </Pressable>
        <Text style={s.countText}>{count}</Text>
        <Pressable onPress={onIncrement} style={s.stepBtn}>
          <Ionicons name="add-circle-outline" size={28} color={colors.textPrimary} />
        </Pressable>
      </View>
    </View>
  );
}
```

### Converting Stepper Counts to PlateBreakdown Format
```typescript
// Convert Record<number, number> to PlateBreakdown format for BarbellDiagram
function countsToBreakdown(plateCounts: Record<number, number>): { weight: number; count: number }[] {
  return Object.entries(plateCounts)
    .filter(([_, count]) => count > 0)
    .map(([weight, count]) => ({ weight: Number(weight), count }))
    .sort((a, b) => b.weight - a.weight);
}
```

### Enhanced Remainder Warning with Missing Plate Info
```typescript
function getMissingPlateMessage(
  remainder: number,
  enabledPlates: number[],
  allPlates: number[],
  unitLabel: string
): string {
  const disabledPlates = allPlates.filter(p => !enabledPlates.includes(p));
  const helpfulPlate = disabledPlates.find(p => p <= remainder);
  if (helpfulPlate) {
    return `Cannot load exactly -- ${helpfulPlate} ${unitLabel} plates not in your inventory`;
  }
  return `Cannot load exactly -- ${remainder} ${unitLabel} unaccounted`;
}
```
